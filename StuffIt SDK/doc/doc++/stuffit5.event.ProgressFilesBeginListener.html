Event handler for the beginning of file count in archive decoding or creation.

This event callback is invoked at the beginning of the decoding phase (in readers) and at the beginning of archive creation (in writers).

It carries the size argument that determines the number of items (files and/or folders) to be decoded or placed into the archive.

This event callback is similar to the ProgressSizeBeginEvent, but begins a sequence of progress updates dealing with item counting in the archive being decoded or created, instead of their sizes.
